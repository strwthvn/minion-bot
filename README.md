# Minion Bot

Discord-бот сообщества **Z-Family**, построенный на Discord.js v14.

Бот недавно появился и активно развивается — у нас много идей и планов. Если хочешь помочь, загляни в [CONTRIBUTING.md](CONTRIBUTING.md).

## Возможности

### Ивенты (`/event`)

Полноценная система управления ивентами с embed-сообщениями в выделенном канале:

- **`/event create`** — создание ивента через модальное окно (название, описание, дата/время МСК, лимит участников, кастомные реакции). После создания можно выбрать роль для пинга.
- **`/event list`** — список предстоящих или прошедших ивентов с пагинацией.
- **`/event edit <id>`** — редактирование ивента (название, описание, дата, лимит, реакции, добавление/удаление участников). Доступно только создателю.
- **`/event cancel <id>`** — отмена ивента с уведомлением участников.

**Регистрация через реакции:** пользователи записываются на ивент реакциями на embed-сообщение. Поддерживаются кастомные реакции (например, роли: танк/хилер/ДД). Реакция ❌ — отказ от участия. При наличии лимита формируется очередь запасных с автоматическим продвижением.

**Напоминания:** автоматические уведомления участникам за 24 часа, 1 час и 5 минут до начала. Автоматическое завершение прошедших ивентов.

### Другие команды

- **`/who <текст>`** — цитирует текст и пингует случайного участника сервера.
- **`/ping`** — проверка работоспособности бота.

### Логирование перемещений в голосовых каналах

Отслеживает принудительные перемещения пользователей в указанный голосовой канал и логирует их в текстовый канал (кто переместил и кого).

## Быстрый старт

### Требования

- [Node.js](https://nodejs.org/) 22+
- Discord-бот, созданный в [Discord Developer Portal](https://discord.com/developers/applications)

### Установка

```bash
git clone https://github.com/strwthvn/minion-bot.git
cd minion-bot
npm install
```

Создайте файл `.env` на основе `.env.example`:

```
DISCORD_TOKEN=токен_бота
DISCORD_CLIENT_ID=id_приложения
DISCORD_GUILD_ID=id_сервера       # опционально, для guild-scoped команд
```

| Переменная | Описание | Обязательная |
|---|---|---|
| `DISCORD_TOKEN` | Токен бота из Developer Portal | Да |
| `DISCORD_CLIENT_ID` | Application ID из Developer Portal | Да |
| `DISCORD_GUILD_ID` | ID сервера для регистрации команд только на нём. Если не указан — команды регистрируются глобально | Нет |

Зарегистрируйте slash-команды:

```bash
npm run deploy
```

### Запуск

```bash
# Разработка с hot-reload
npm run dev

# Обычный запуск
npm start

# Docker
docker compose up -d
```

## Структура проекта

```
src/
├── index.js              # Точка входа, загрузка команд, роутинг событий
├── config.js             # ID каналов, цвета, настройки
├── deploy-commands.js    # Регистрация slash-команд через Discord API
├── commands/             # Модули команд (auto-load)
│   ├── event.js          # Управление ивентами
│   ├── who.js            # Случайный пинг
│   └── ping.js           # Пинг
├── handlers/
│   ├── reactionHandler.js    # Обработка реакций на ивенты
│   └── voiceMoveHandler.js   # Логирование перемещений в войсе
├── services/
│   ├── eventService.js       # CRUD ивентов
│   ├── participantService.js # Управление участниками
│   ├── embedService.js       # Формирование embed-сообщений
│   └── reminderService.js    # Автоматические напоминания
├── database/
│   └── connection.js         # Подключение к SQLite, схема таблиц
└── utils/
    └── dateParser.js         # Парсинг дат в МСК
```

## Технологии

- **Node.js 22** + **Discord.js v14**
- **better-sqlite3** — хранение ивентов и участников
- **Docker** — контейнеризация и деплой
